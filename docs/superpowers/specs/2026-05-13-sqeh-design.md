# SQEH Design: Spiking Quaternion Event Hypergraph

## Purpose

SQEH is a new irregular multivariate time-series hypergraph model whose hyperedges are latent event processes, not fixed time or variable labels. The primary goal is paper-level novelty. The secondary goal is to remain competitive with, and ideally outperform, HyperIMTS on sparse and dense datasets.

The design replaces PSQH's static prototype windows with event-triggered dynamic hyperedges. This avoids degenerating into a soft version of HyperIMTS timestep hyperedges.

## Non-Goals

- Do not preserve PSQH's fixed `t_center + Gaussian window` as the primary routing mechanism.
- Do not build a HyperIMTS variant with added quaternion or spike residuals.
- Do not maximize model complexity in the first version. SQEH v1 should have a small number of attributable mechanisms.
- Do not claim superiority unless multi-seed experiments support it.

## Core Claim

HyperIMTS uses label-defined static hyperedges. PSQH uses learned prototypes with static temporal windows. SQEH uses event-triggered dynamic hyperedges whose topology is generated by spike state, quaternion relation routing, and density-adaptive fields.

```text
HyperIMTS: H[n,k] = 1[var_id/time_id matches hyperedge k]
PSQH:     H[n,k] = prototype_affinity(q_n, p_k) * fixed_window(t_n, c_k)
SQEH:     H[n,k] = event_spike_k(t_n, history) * quat_relation(q_n, e_k) * density_field(n,k)
```

## Model-Level Innovations

### 1. Event-Spiking Hyperedges

Each hyperedge is a latent event unit with state. It receives input from observed cells, integrates evidence over time, fires when its membrane crosses a threshold, then enters a refractory period.

For event hyperedge `k` at timestep `t`:

```text
I_k(t) = aggregate_n route_pre[n,k] * q_n, for observed nodes at t
u_k(t) = decay_k * u_k(t-1) + input_proj_k(I_k(t))
s_k(t) = surrogate_step(u_k(t) - theta_k)
u_k(t) = u_k(t) - s_k(t) * theta_k
r_k(t+1) = refractory_update(r_k(t), s_k(t))
```

`s_k(t)` is the event spike. It controls whether hyperedge `k` is active for nodes near timestep `t`.

This makes hyperedges dynamic event processes rather than static prototype buckets.

### 2. Quaternion Relation Routing

Nodes and event hyperedges live in quaternion space. Routing is based on their relative Hamilton relation:

```text
r_nk = q_n ⊗ conj(e_k)
relation_logit[n,k] = MLP([Re(r_nk), ||Im(r_nk)||, ||r_nk||])
relation_gate[n,k] = sigmoid(relation_logit[n,k])
```

The event-to-node message also uses the relation:

```text
msg_n = sum_k H[n,k] * QLinear(e_k ⊗ r_nk)
```

Quaternion is therefore used for both topology generation and message transformation. It is not only a four-way feature split.

### 3. Density-Adaptive Event Field

SQEH estimates local observation density for each cell and uses it to adapt event receptive fields.

Inputs:

- local temporal density around `t_n`
- variable-level observation density for `v_n`
- local inter-observation gap

The field gate is:

```text
density_feat_n = [rho_time_n, rho_var_n, delta_t_prev_n, delta_t_next_n]
bw_nk = softplus(base_bw_k + field_mlp(density_feat_n)_k)
field_gate[n,k] = exp(-distance_to_recent_event[n,k]^2 / bw_nk^2)
```

The field is not a fixed absolute-time Gaussian window. It is conditioned on sample density and recent event activity. Sparse regions get wider support; dense regions stay sharper.

## Architecture

### Input Representation

The input follows existing PyOmniTS/HyperIMTS conventions:

- `x`: `[B, seq_len, V]`
- `x_mask`: `[B, seq_len, V]`
- `y`: `[B, pred_len, V]`
- `y_mask`: `[B, pred_len, V]`

The model flattens observed cells into:

- `value`: `[B, N]`
- `time_idx`: `[B, N]`
- `time_norm`: `[B, N]`
- `var_idx`: `[B, N]`
- `mask`: `[B, N]`

### Cell Encoder

Use a quaternion cell encoder similar to PSQH's stable encoder:

```text
q_n = QuaternionCellEncoder(value_n, time_norm_n, var_idx_n, spike=mask_n)
```

`q_n` has shape `[B, N, D]`, where `D % 4 == 0`.

### Event-Spike Router

Each SQEH block owns `K` event hyperedges:

- event state `e_k`: `[K, D]`
- membrane state `u_k`: per batch, `[B, K]`
- threshold `theta_k`: `[K]`
- decay `decay_k`: `[K]`
- refractory counter: per batch, `[B, K]`

The router builds per-timestep event inputs by scattering node evidence to timesteps:

```text
pre_route[n,k] = sigmoid(pre_mlp(quat_relation(q_n, e_k)))
t_input[t,k] = scatter_sum(pre_route[n,k] * value_proj(q_n), time_idx=n.t)
```

Then it scans timesteps in order and emits event spikes `s[t,k]`.

The v1 implementation uses a differentiable surrogate spike:

```text
s = hard_step(u - theta) + sigmoid(alpha * (u - theta)) - detach(sigmoid(alpha * (u - theta)))
```

This provides binary forward behavior and usable gradients.

### Incidence Matrix

For node `n` at timestep `t_n`:

```text
event_gate[n,k] = gather(s[t_n,k])
relation_gate[n,k] = sigmoid(relation_logit(q_n, e_k))
field_gate[n,k] = density_adaptive_field(n,k)
H[n,k] = mask[n] * event_gate[n,k] * relation_gate[n,k] * field_gate[n,k]
```

Normalize only after preserving total event intensity:

```text
H_norm[n,k] = H[n,k] / clamp(sum_k H[n,k], min=eps)
```

Use raw `H` for node-to-event aggregation so event intensity remains meaningful. Use `H_norm` only for the final event-to-node weighted sum so node updates remain numerically stable. The default v1 avoids mandatory per-node softmax over all `K`, because PSQH showed that zero-sum competition can cause prototype death and route collapse.

### Hypergraph Message Passing

Node to event:

```text
h_k = sum_n H[n,k] * q_n / clamp(sum_n H[n,k], eps)
e_k' = EventUpdate(e_k, h_k)
```

Event update should be lightweight:

- quaternion linear transform for event input
- event self-attention across `K` events
- residual and quaternion layer norm

Event to node:

```text
r_nk = q_n ⊗ conj(e_k')
m_nk = QuaternionLinear(e_k' ⊗ r_nk)
msg_n = sum_k H_norm[n,k] * m_nk
q_n' = QuaternionLayerNorm(q_n + dropout(msg_n))
```

Optional node self-attention remains enabled by default for dense datasets, because PSQH showed it improves HumanActivity.

### Decoder

Use the stable decoder pattern from PSQH, but keep it outside the novelty claim:

```text
t_ctx = scatter_mean(q, time_idx)
v_ctx = scatter_mean(q, var_idx)
pred_n = MLP(q_n + alpha_t * t_ctx[t_n] + alpha_v * v_ctx[v_n])
```

The decoder is a stabilizer. The paper narrative should emphasize that topology and message passing are generated by event-spiking quaternion hyperedges.

## Diagnostics

SQEH must log enough information to avoid PSQH's "initialization already solved it" failure mode:

- event spike rate per layer
- alive event ratio: fraction of events with nonzero activation
- mean and max membrane value
- threshold mean and standard deviation
- refractory active ratio
- `H` density and entropy
- relation gate mean and standard deviation
- field bandwidth mean by local density bucket
- gradient norms for event state, threshold, decay, relation MLP, and field MLP

## Ablations

Minimum paper-facing ablations:

- `no_spike`: replace event spikes with continuous relation gates.
- `no_quat_relation`: replace Hamilton relation with dot-product routing.
- `static_field`: replace density-adaptive field with fixed bandwidth.
- `softmax_route`: force per-node softmax routing to test PSQH-style competition.
- `no_node_attn`: remove node self-attention stabilizer.
- `no_decoder_ctx`: remove scatter time/variable decoder context.

These ablations isolate the three main innovations plus the two stabilizers.

## Experiment Plan

Initial smoke datasets:

- synthetic small batch for shape, gradient, and finite output tests
- USHCN for sparse-event advantage
- HumanActivity for dense long-sequence stress
- P12 for dense clinical irregularity

First-pass target:

- SQEH should run without NaN/OOM on CPU smoke and one small GPU run.
- At least one dataset should match PSQH's best single-seed range before expanding.
- If HumanActivity regresses badly, inspect event spike collapse before adding modules.

Final evaluation:

- compare against HyperIMTS, PSQH, and QSHNet-like references
- use at least 5 seeds for any claim of improvement
- report mean and standard deviation
- report diagnostic evidence that events are active and differentiated

## Success Criteria

Novelty success:

- Hyperedges are event-triggered, not static labels or static time-window prototypes.
- Quaternion relation controls both incidence and message passing.
- Density-adaptive event fields change routing behavior across sparse and dense regions.

Engineering success:

- `tests/sqeh_smoke.py` passes.
- `models/SQEH.py` supports train, validation, and test stages through the existing PyOmniTS interface.
- `MSE_aux` can consume the model output.
- Configs and scripts exist for USHCN, P12, and HumanActivity.

Empirical success:

- USHCN should remain clearly competitive because event sparsity is the model's natural regime.
- HumanActivity and P12 should not collapse relative to PSQH.
- Any "beats HyperIMTS" claim must use multi-seed evidence.

## Implementation Boundary for v1

Implement only:

- `models/SQEH.py`
- `tests/sqeh_smoke.py`
- config arguments in `utils/configs.py` and `utils/ExpConfigs.py`
- scripts/configs for USHCN, P12, HumanActivity

Do not modify HyperIMTS, PSQH, or QSHNet internals for SQEH v1.

## v1 Defaults

- `K`: 64 for USHCN and P12, 128 for HumanActivity.
- Number of SQEH blocks: 2.
- Spike surrogate slope: 10.0.
- Refractory length: 2 timesteps.
- Density field inputs: local temporal density, previous observation gap, and next observation gap. Variable-level density is excluded from v1 to keep the first implementation identifiable.
- Route normalization: use raw `H` for event intensity and row-normalized `H_norm` only for the final event-to-node weighted sum. Do not use per-node softmax unless running the `softmax_route` ablation.
