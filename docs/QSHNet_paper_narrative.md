## 当前推荐标题方向

当前更适合的标题方向是：

- 基于超图、四元数增强与事件感知注入的时间序列统一建模框架
- 面向不规则多变量时间序列的超图-四元数-事件统一建模方法

当前不建议在标题中直接使用以下强表述：

- 「稳定协同」
- 「三分支一致增益」
- 「事件机制显著提升所有数据集性能」

原因很直接：这些说法超出了现有实验能够稳定支撑的范围。

## 当前最安全的核心叙事

### 1. 三元素框架的角色分工

当前最安全、也最符合实验事实的统一叙事是：

- **超图（Hypergraph）：** 作为结构主干，负责不规则多变量观测之间的高阶关系建模。
- **四元数增强（Quaternion）：** 作为主要增强分支，在不破坏主干表达的前提下建模更丰富的跨特征耦合。
- **事件感知注入（Spike / Event）：** 作为事件敏感的辅助注入支路，负责将偏离变量上下文的观测扰动，以受控方式注入 hyperedge 更新。

这个表述的关键点在于：

- 三者都属于统一框架；
- 但三者不是「完全对等」的收益来源；
- 其中超图是主干，四元数是主增强，事件支路是事件感知注入机制。

### 2. 当前可以被证据支撑的主结论

截至 2026-04-16，当前可以安全使用的主结论是：

1. 旧版 M1 的真实训练动力学并不是预期中的「Spike + Event + Quaternion 协同」。
2. 旧版 M1 更接近 `HyperIMTS + retain 调制 + quaternion 残差`。
3. 通过结构性改动，`event` 分支已经从最初的死分支变成可训练、可控、真实参与训练的支路。
4. `event` 支路一旦被激活，如果缺少独立控制，会在高方差数据上放大尾部不稳定性。
5. 为 `event` 支路增加独立归一化后，可以在保留其参与性的同时，显著改善短重复下的稳定性。

### 3. 当前不能直接写成既定事实的说法

以下说法目前不建议直接写进摘要、引言或结论：

1. `event` 机制已经带来稳定且普适的跨数据集收益。
2. 三元素分支已经形成稳定协同。
3. 当前版本已经是最终主线定版。

这些说法不安全的原因是：

- `eventnorm_main` 在 `USHCN itr=5` 上表现很好，但 `itr=10` 仍暴露出尾部风险；
- 因此更准确的结论应当是「可训练、可控，但尚未彻底稳定」。

## 推荐的实验叙事链

论文中更适合采用「诊断驱动的结构修正」这条叙事链，而不是简单罗列调参过程。

### 阶段一：发现旧 M1 的真实动力学

通过 `QSHDiag` 与本地参数诊断，首先发现：

- `event_proj.weight norm = 0.0`
- `event_residual_scale = 0.0`
- 真正持续学习的是 `retain` 与 `quaternion`

因此得到第一个关键结论：

> 旧 M1 虽然结构上包含 Spike / Event / Quaternion 三部分，但真实训练动力学并没有把 `event` 支路激活起来。

这一步是后续所有结构试验的逻辑起点。

### 阶段二：先确认当前真正活跃的稳定化对象

在方向 B 中，先承认当前真实活跃结构是 `retain + quaternion`，并做最小稳定化：

- `retaincap_main`：只约束 `retain`
- `retaincap_quatbound`：在 `retain cap` 基础上再限制 quaternion residual

结果显示：

- `retaincap_main` 在 `USHCN itr=5` 上达到 `0.1673 ± 0.0033`
- `retaincap_quatbound` 没有带来额外核心收益

这说明：

> 旧 M1 的坏轮确实与活跃分支放大有关，但简单约束 quaternion residual 不是关键收益来源。

### 阶段三：围绕统一框架叙事重新激活 `event` 分支

在明确了旧 M1 的真实动力学后，再回到方向 A，目标不再是「继续调 `event gate`」，而是：

> 先让 `event` 分支真正参与训练，再讨论其对统一框架的贡献。

#### 试验 A1：`eventfusion_sigmoid`

只增强 `event` 融合强度表达：

- 将原先近零残差语义改为 `sigmoid` 受控融合；
- 初始化到 `sigmoid ≈ 0.1`。

结果：

- `event` 分支首次真实参与训练；
- `HumanActivity itr=3` 稳定；
- `USHCN itr=5` 退化到 `0.1803 ± 0.0386`。

这说明：

> 让 `event` 活起来是可行的，但单纯增强融合强度会重新放大高方差尾部风险。

#### 试验 A2：`eventnorm_main`

在 `eventfusion_sigmoid` 基础上，只新增 `event` 支路的独立归一化：

- `temporal_event_norm`
- `variable_event_norm`

结果：

- `HumanActivity itr=3`：`0.0430 ± 0.0013`
- `USHCN itr=5`：`0.1653 ± 0.0011`

这一步给出的核心证据是：

> `event` 支路的问题不只是「强度不足」，更重要的是缺少独立控制；加入独立归一化后，`event` 可以在保持参与性的同时显著改善短重复稳定性。

#### 试验 A2 扩展：`eventnorm_itr10`

为了确认 `itr=5` 结果不是偶然，又对 `USHCN` 补做了 `itr=10`。

结果：

- `USHCN itr=10`：`0.1829 ± 0.0279`
- 最优：`0.1628`
- 最差：`0.2353`

这一步进一步收紧了结论边界：

> `eventnorm` 版本已经足以支撑统一框架叙事，但长重复下仍有尾部不稳定性，因此更适合作为「当前主线候选」而不是「最终定版」。

#### 试验 A2.0：失败分支与归因收窄

在 `eventnorm_main` 之后，又连续排除了 3 条看似合理、但实际无效的后续方向：

1. `eventgain_main`
   - 额外加入主干条件化 gain；
   - `HumanActivity itr=3` 为 `0.0424 ± 0.0006`；
   - `USHCN itr=5` 退化到 `0.1974 ± 0.0633`。

2. `event_temporal_only`
   - 只保留 temporal `event` 注入；
   - `HumanActivity itr=3` 为 `0.0422 ± 0.0005`；
   - `USHCN itr=5` 为 `0.1779 ± 0.0365`。

3. `eventnorm_clip`
   - 对归一化后的 `event_delta` 做 `tanh` 裁剪；
   - `HumanActivity itr=3` 为 `0.0422 ± 0.0005`；
   - `USHCN itr=5` 退化到 `0.1898 ± 0.0381`。

这三步的共同意义不是“又多做了几次失败实验”，而是把问题边界明显收窄了：

> `event` 支路当前最需要的不是额外条件化控制器、不是删掉一条注入支路、也不是直接裁剪 `event_delta` 形状，而是更平滑地限制总注入量。

#### 试验 A2.1：`eventscalecap_main`

在 `eventnorm_main` 基础上，不改拓扑，只对 `event_scale` 引入温和上界：

- `event_scale = clamp(sigmoid(event_residual_scale), max=0.12)`

结果：

- `HumanActivity itr=3`：`0.0430 ± 0.0013`
- `USHCN itr=5`：`0.1663 ± 0.0027`

这一步的作用是：

> 验证“控制总注入量”是一条安全方向。它不会像 `eventnorm_clip` 一样制造两极分化，也不会像 `eventgain_main` 一样明显恶化 `USHCN`。

#### 试验 A2.1 扩展：`eventscalecap_itr10`

为了判断 `event_scale` 温和上界是否对长重复稳定性真正有帮助，又补做了 `USHCN itr=10`。

结果：

- `USHCN itr=10`：`0.1728 ± 0.0222`
- 最优：`0.1610`
- 最差：`0.2355`

与 `eventnorm_itr10` 的 `0.1829 ± 0.0279` 相比，这一步的意义非常明确：

> 对 `event` 总注入量做轻量上界约束，能够改善长重复下的均值与方差；但由于尾部坏轮仍然存在，它更适合被定义为“当前更好的主线候选”，而不是“已经彻底稳定的最终版本”。

## 当前最适合写入论文的表述模板

### 摘要 / 引言中的安全写法

推荐写法：

> 我们提出一种统一框架，将超图结构建模、四元数增强与事件感知注入有机结合，用于不规则多变量时间序列建模。实验与诊断分析表明，超图主干与四元数增强构成主要表达基础，而事件支路在经过独立控制后可作为真实可训练的事件注入机制参与模型更新。

不推荐写法：

> 三个模块在所有数据集上稳定协同并带来一致收益。

### 方法部分中的安全写法

推荐写法：

> 事件支路并不替代主干消息传递，而是作为一条受控的辅助注入路径，将偏离变量上下文的观测信号以归一化后的增量形式注入 temporal / variable hyperedge 更新。

在最新版本中，这条“受控辅助路径”的控制还进一步细化为：

> 对 `event` 总注入量引入温和上界，以缓解高方差数据上的尾部放大。

这个表述能够准确反映当前实现，也和实验观察一致。

### 实验结论中的安全写法

推荐写法：

> 结构诊断表明，若缺少独立控制，激活后的事件支路会在高方差数据上放大尾部不稳定性；而在引入事件支路独立归一化后，再进一步对事件总注入量施加温和上界，模型能够在保留事件机制参与性的同时改善长重复实验中的稳定性。

这个说法既保住了三元素统一框架，也没有过度承诺长期稳定性。

### 方法动机段（可直接用于引言 / 方法开头）

不规则多变量时间序列建模同时面临 3 类耦合难点：观测节点之间存在高阶结构依赖，不同变量与时间上下文之间存在复杂的跨维交互，而局部异常或偏离上下文的事件信号又往往只在少量观测上短暂出现。现有基于 HyperIMTS 的超图建模框架能够较好地刻画不规则观测之间的结构关系，但其主干更新仍主要依赖线性映射，对更高阶的跨特征耦合以及事件敏感注入缺乏显式建模。基于这一观察，我们构建了一个由超图主干、四元数增强与事件感知注入组成的统一框架。其核心思想是：以超图结构作为基础消息传递骨架，以四元数残差增强跨特征耦合表达，并将偏离变量上下文的观测扰动作为一条受控的事件注入支路，参与 temporal / variable hyperedge 的更新。进一步地，我们并不假设事件支路天然稳定，而是通过诊断驱动的结构修正，对其注入方式进行独立控制，使其在保留事件敏感性的同时尽量降低高方差数据上的不稳定放大。

### 实验结论段（可直接用于实验总结 / Discussion）

实验结果表明，QSH-Net 的主要表达基础仍由超图主干与四元数增强构成，而事件支路能否真正发挥作用，取决于其是否被激活以及是否具备独立控制。对旧版 M1 的参数诊断显示，`event_proj` 与 `event_residual_scale` 在训练中长期保持静默，说明早期版本虽然结构上包含事件分支，但真实训练动力学并未使其参与模型更新。围绕这一问题，我们进一步开展了结构性试验：单纯增强事件融合强度虽然能够激活事件支路，但会在 USHCN 这类高方差数据上显著放大尾部风险；而在事件支路上引入独立归一化后，模型能够在保留事件参与性的同时显著改善短重复实验中的稳定性。需要指出的是，`eventnorm` 版本在 `USHCN itr=10` 上仍然暴露出尾部不稳定性，因此当前更合理的结论不是「三分支已经稳定协同」，而是 QSH-Net 已经形成一个由超图主干、四元数增强与事件感知注入组成的统一框架，其中事件支路已从静默模块转变为真实可训练、可控但尚未彻底稳定的结构组成部分。

### 贡献表述段（可直接用于引言末尾 / 贡献总结）

基于当前证据，更稳妥的贡献总结可写为：

1. 我们提出一个面向不规则多变量时间序列的统一建模框架，将超图结构建模、四元数增强与事件感知注入有机结合到同一主干中。
2. 我们通过诊断驱动的结构分析揭示了早期 QSH-Net 中事件支路长期静默的真实训练动力学，并说明了模型不稳定性主要来自活跃分支在高方差数据上的放大，而非简单的常规超参数失配。
3. 我们进一步提出对事件注入支路的受控激活与独立归一化机制，使事件分支从挂名模块转变为真实可训练、可控参与模型更新的结构组成部分。
